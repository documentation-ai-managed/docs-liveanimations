## Overview

Liveanimations supports seamless integrations with popular third-party services, webhooks for real-time automation, a robust API for custom solutions, and exports to external platforms. You can extend your documentation workflow by connecting to tools like Slack, GitHub, and Zapier. These features enable automated updates, notifications, and data syncing.

<Callout kind="tip">
  Start with webhooks for quick automations, then explore the API for deeper customizations.
</Callout>

## Webhook Configurations

Set up webhooks to trigger actions on doc events like publishes or edits. Navigate to `https://dashboard.example.com/webhooks` and add a new webhook.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Enter your endpoint URL, such as `https://your-webhook-url.com/liveanimations`.
  </Step>
  <Step title="Select Events" icon="zap">
    Choose events: `doc.published`, `doc.updated`, `user.invited`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and save the configuration.
  </Step>
</Steps>

Handle incoming webhooks with this example payload:

<CodeGroup tabs="Node.js,Python">
  ```javascript
  const express = require('express');
  const app = express();
  app.use(express.json());

  app.post('/liveanimations', (req, res) => {
    const { event, doc_id, user_id } = req.body;
    if (event === 'doc.published') {
      console.log(`Doc ${doc_id} published by ${user_id}`);
    }
    res.status(200).send('OK');
  });

  app.listen(3000);
  ```
  ```python
  from flask import Flask, request, jsonify

  app = Flask(__name__)

  @app.route('/liveanimations', methods=['POST'])
  def webhook():
      data = request.json
      event = data.get('event')
      doc_id = data.get('doc_id')
      user_id = data.get('user_id')
      if event == 'doc.published':
          print(f'Doc {doc_id} published by {user_id}')
      return jsonify({'status': 'OK'}), 200

  if __name__ == '__main__':
      app.run(port=3000)
  ```
</CodeGroup>

## API Access for Custom Integrations

Build custom solutions using the Liveanimations API at `https://api.example.com/v1`. Authenticate with your API key.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `{YOUR_API_KEY}`.
</ParamField>

<ParamField path="docs" param-type="string" required="false">
  Filter docs by ID or slug.
</ParamField>

Fetch a list of docs:

<Request tabs="cURL,JavaScript">
  ```bash
  curl -X GET 'https://api.example.com/v1/docs' \
    -H 'Authorization: Bearer YOUR_API_KEY'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/docs', {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  });
  const docs = await response.json();
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "docs": [
      {
        "id": "doc_123",
        "title": "Getting Started",
        "slug": "getting-started",
        "updated_at": "2024-10-15T10:00:00Z"
      }
    ]
  }
  ```
</Response>

## Exporting Docs to External Platforms

Export your documentation in various formats for sharing or migration.

<Tabs>
  <Tab title="PDF" icon="file-text">
    Generate PDF exports from the dashboard. Ideal for printable guides.
    
    ```
    POST https://api.example.com/v1/docs/{id}/export/pdf
    ```
  </Tab>
  <Tab title="Markdown" icon="code">
    Download raw Markdown files for GitHub or static sites.
    
    ```
    POST https://api.example.com/v1/docs/{id}/export/md
    ```
  </Tab>
  <Tab title="HTML" icon="globe">
    Export as static HTML for embedding on websites.
    
    ```
    POST https://api.example.com/v1/docs/{id}/export/html
    ```
  </Tab>
</Tabs>

<Expandable title="Advanced Export Options">
  Use query params like `?include_comments=true` or `?version=latest` for customized exports.
</Expandable>
